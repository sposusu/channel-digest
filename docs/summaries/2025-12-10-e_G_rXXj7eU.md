# 介紹 Langsmith fetch：將 Langsmith 資料帶入您的開發工作流程

## 重點摘要
Langsmith fetch 是一個新的命令列工具，可以將 Langsmith 專案中的追蹤資料下載到您的本機檔案系統。這讓 Claude Code 或 Deep Agent CLI 等編碼 AI 助理能直接存取並分析 agent 的運行資料，從而自動診斷錯誤並提出改善建議。

## 故事大綱
- **開場**：影片開頭介紹了 `Langsmith fetch`，一個旨在解決分析日益複雜的 AI agent（具有大量工具呼叫和長提示詞）追蹤紀錄的難題。目標是讓編碼助理能夠存取這些資料來協助開發。
- **中段**：講者展示了一個實際範例。首先，他運行一個「深度研究 agent」來產生 Langsmith 追蹤紀錄。接著，他啟動一個 AI 編碼助理 (Claude)，並指示它使用 `langsmith fetch` 來抓取剛剛那個專案的資料。AI 助理成功下載了兩次運行的執行緒，並對其進行分析，點出了兩次運行結果不一致等關鍵觀察。
- **結尾**：基於分析，AI 助理提出具體的改善建議，例如「增加查詢消歧義」、「加入來源驗證」和「建立簡潔的摘要格式」。影片總結，`Langsmith fetch` 能將 Langsmith 中豐富的資料帶入終端機，讓編碼助理化身為您的專屬 AI 工程師。

## 關鍵見解
1.  **解決手動除錯的痛點**：隨著 agent 變得越來越複雜，手動檢查大量的追蹤紀錄非常耗時。`Langsmith fetch` 提供了一個自動化的解決方案。
2.  **賦予 AI 助理新能力**：它讓編碼助理不再只是一個程式碼產生器，而是能理解應用程式運行狀況並提出建議的「AI 工程師」。
3.  **資料驅動的開發**：開發者可以利用 AI 助理，根據實際的運行資料來分析問題，而不是僅僅依靠猜測。
4.  **具體的改善建議**：在此範例中，AI 助理不僅發現了問題（不一致、冗長），還提出了清晰的解決方案（查詢消歧義、簡潔摘要）。
5.  **無縫整合工作流程**：此工具能輕易地整合到現有的開發環境中，只需透過 `pip install` 安裝並在終端機中與 AI 助理互動即可。

## 精彩時刻
- **核心指令**：`use linksmith fetch to analyze the most recent threads in [project ID]`，這個簡單的提示詞就啟動了整個分析流程。
- **AI 的自主分析**：看著 AI 助理自主操作 CLI、建立目錄、下載資料，然後產出詳細的分析報告，非常令人印象深刻。
- **價值主張**：影片結尾有力地總結了其核心價值：「將 Langsmith 中所有豐富的資料帶入您的終端機，帶入您的編碼助理，讓它們為您成為 AI 工程師。」

# Introducing Langsmith Fetch: Bring Langsmith Data into Your Development Workflow

## TL;DR
Langsmith fetch is a new command-line utility for pulling trace data from Langsmith projects to your local filesystem. This empowers coding AI assistants like Claude Code or Deep Agent CLI to directly access and analyze agent run data, allowing them to automatically diagnose errors and suggest improvements.

## Story Flow
- **Beginning**: The video introduces `Langsmith fetch`, a tool designed to solve the challenge of analyzing traces from increasingly complex AI agents (with many tool calls and long prompts). The goal is to give coding assistants access to this data to aid development.
- **Middle**: The presenter demonstrates a practical example. First, a "deep research agent" is run to generate traces in Langsmith. Then, he starts a coding assistant (Claude) and instructs it to use `langsmith fetch` to pull the data from that project. The AI assistant successfully downloads the threads from two runs, analyzes them, and highlights key observations, such as inconsistencies between the outputs.
- **End**: Based on its analysis, the AI assistant provides concrete recommendations for improvement, such as "add better query disambiguation" and "add a concise summary format." The video concludes that `Langsmith fetch` brings the rich data from Langsmith into the terminal, turning coding assistants into expert AI engineers for you.

## Key Insights
1.  **Solves Manual Debugging Pain**: As agents become more complex, manually inspecting long traces is time-consuming. `Langsmith fetch` provides an automated alternative.
2.  **Empowers AI Assistants**: It elevates a coding assistant from a simple code generator to an "AI engineer" that can understand an application's runtime behavior and suggest improvements.
3.  **Data-Driven Development**: Developers can leverage an AI assistant to analyze issues based on actual run data, rather than relying on guesswork.
4.  **Concrete Improvement Suggestions**: In the demo, the assistant didn't just find problems (inconsistency, verbosity) but also proposed clear solutions (query disambiguation, concise summaries).
5.  **Seamless Workflow Integration**: The tool is easily integrated into an existing development environment via a `pip install` and interaction with an AI assistant in the terminal.

## Notable Moments
- **The Core Command**: The simple prompt `use linksmith fetch to analyze the most recent threads in [project ID]` kicked off the entire analysis workflow.
- **The AI's Autonomous Analysis**: It was impressive to watch the AI assistant autonomously operate the CLI, create a directory, download the data, and then produce a detailed analysis report.
- **The Value Proposition**: The video's powerful closing statement on its core value: "to bring all the rich data that's in Langsmith into your terminal, into your coding agents so that they can become AI engineers for you."
