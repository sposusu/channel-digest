# Anthropic 的「技能」概念：擴展 AI 代理能力的新範式

## 重點摘要
「技能」(Skills) 是一種為 AI 代理提供標準作業流程 (SOP) 的新方法。它讓代理程式透過讀取簡單的 Markdown 指令文件，並利用少數核心工具（如檔案系統存取、Shell 命令）來完成複雜任務，從而有效擴展其能力，同時避免了傳統工具調用層的臃腫和混亂。

## 故事大綱
- **開場**：影片主講人 Lance 介紹了 Anthropic 提出的「技能」概念，並立即展示他的 Deep Agent CLI 如何利用一項「網頁研究」技能，來自動化執行關於「情境工程」(Context Engineering) 的研究任務。
- **中段**：Lance 解釋了「技能」背後的哲學。每個技能是一個包含 `skill.md` 檔案的資料夾，該檔案的 YAML 前言部分會被預先載入代理的系統提示中（稱為「漸進式揭露」），讓代理知道有哪些技能可用。只有當任務觸發某個技能時，代理才會去讀取完整的 Markdown 指令內容。他強調，這種方法遠比為模型綁定大量函式更優越，因為它能減少模型的認知負擔和在選擇工具時的潛在混淆。
- **結尾**：他展示了第二個範例：要求代理建立一個 LangGraph 代理。代理程式使用了「LangGraph 文件」技能，從一個文字檔中讀取相關的文件 URL，抓取內容後，成功編寫出 `agent.py` 程式碼。他總結道，「技能」是一個簡單而強大的概念，只要給予代理檔案系統的存取權，就能輕鬆擴展其行動空間。

## 關鍵見解
- **擴展行動空間**：「技能」能在不增加工具調用層複雜性的情況下，顯著擴展代理的行動能力。
- **漸進式揭露 (Progressive Disclosure)**：僅預先載入技能的簡短摘要（YAML 前言），待需要時才讀取完整內容。這種模式非常高效，可節省大量的 token 並讓代理專注於當前任務。
- **原子化工具的力量**：一個擁有少數原子化原生工具（如 Shell、檔案讀寫）並能存取檔案系統的代理，比一個擁有數百個專門工具的代理更強大、更不易混淆。
- **易於客製化**：「技能」就像是給 AI 的標準作業流程，可以用簡單的 Markdown 語言編寫，讓使用者能輕鬆地為代理定義新的工作流程。
- **關鍵依賴**：此架構要成功運作，最關鍵的前提是必須給予代理程式存取電腦檔案系統的權限。

## 精彩時刻
- 開頭的現場演示：代理自動識別並執行「網頁研究」技能，整個過程流暢自然，完美展示了其自主性。
- 犀利的觀點：「有些人甚至吹噓『我的代理有 100 個工具』，但你會發現更先進的代理...實際上工具數量很少。」
- 「漸進式揭露」的解釋，清楚地點出了為何此方法在節省成本和提升效率上如此卓越。
- 結語：「一個非常簡單的概念，一種擴展代理行動空間的絕佳方式，且無需工具調用的複雜性。」

# Anthropic's "Skills": A New Paradigm for Expanding AI Agent Capabilities

## TL;DR
"Skills" are a new way to provide AI agents with Standard Operating Procedures (SOPs). This approach allows an agent to perform complex tasks by reading simple Markdown instruction files and using a few core tools (like file system access and shell commands), effectively expanding its abilities while avoiding the bloat and confusion of a traditional tool-calling layer.

## Story Flow
- **Beginning**: The speaker, Lance, introduces "Skills," a concept from Anthropic, and immediately demonstrates how his Deep Agent CLI uses a "web research" skill to autonomously execute a research task on "Context Engineering."
- **Middle**: Lance explains the philosophy behind Skills. Each skill is a directory with a `skill.md` file whose YAML front matter is pre-loaded into the agent's system prompt (a concept called "progressive disclosure"). This makes the agent aware of available skills without being overwhelmed. Only when a task triggers a specific skill does the agent read the full Markdown instructions. He argues this is superior to binding many functions to a model, as it reduces cognitive load and potential confusion when selecting tools.
- **End**: He shows a second example, asking the agent to create a LangGraph agent. The agent uses a "LangGraph docs" skill to read relevant documentation URLs from a text file, fetches the content, and successfully writes the `agent.py` code. He concludes that Skills are a simple yet powerful concept for expanding an agent's action space, provided it has file system access.

## Key Insights
- **Expanding Action Space**: Skills significantly expand an agent's capabilities without bloating the function-calling layer.
- **Progressive Disclosure**: Loading only short skill summaries (YAML front matter) initially and reading full instructions on-demand is highly efficient, saving significant tokens and keeping the agent focused.
- **The Power of Atomic Tools**: An agent with a few atomic, native tools (like shell, file I/O) and file system access is more powerful and less prone to confusion than an agent with hundreds of specialized tools.
- **Easy Customization**: Skills act as SOPs for AI that can be written in simple Markdown, making it easy for users to define new workflows for the agent.
- **The Critical Dependency**: The key prerequisite for this architecture to work is giving the agent access to a computer's file system.

## Notable Moments
- The live demo at the start: The agent seamlessly finds and executes the `web_research` skill, perfectly showcasing its autonomy.
- The sharp insight: "some people even flex that 'oh okay my agent has 100 tools' what you're seeing with more state-of-the-art agents... is actually a small number of tools."
- The explanation of "progressive disclosure," which clearly highlights why this method is so superior for cost-savings and efficiency.
- The closing statement: "A very simple concept, a very nice way to expand the action space of your agent and doesn't require the complexities of tool calling."
