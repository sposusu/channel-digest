# Langsmith Agent Builder 公開測試版發布與實作展示

## 重點摘要
Langsmith 推出無程式碼的 Agent Builder，讓使用者能透過聊天方式建立、測試和迭代具備工具、記憶和子代理人的 AI 代理人。此工具旨在將模糊的想法轉化為可靠的自動化工具，簡化從深度研究到個人助理等複雜任務。

## 故事大綱
- **開場：** 影片宣布 Langsmith Agent Builder 進入公開測試階段。它被定位為一個真正的「代理人建構器」，而非傳統的「工作流程建構器」，其核心由代理人自身的提示、工具和記憶所驅動。
- **中段：** 影片核心為一個完整的建立流程，目標是打造一個「LinkedIn 招募代理人」。
    1.  **建立與釐清**：使用者僅提供模糊指令，建構器會主動研究並反問數個問題，以釐清代理人的具體任務、觸發方式和所需資料。
    2.  **測試與學習**：在「除錯模式」下，代理人初次嘗試直接搜尋，但被使用者即時糾正，並被教導「你應該先問後續問題以更了解工作描述。記住這一點。」代理人隨即將此指令更新到自己的系統提示中，實現了真正的「記憶」。
    3.  **實戰應用**：展示一個成熟的代理人如何處理招募任務。它先詢問細節，檢查檔案系統中是否已存有相關的工作描述，進行小規模的「校準搜尋」以獲得使用者回饋，最後在確認方向正確後，才大規模抓取 50 位候選人並將結果存成檔案。
- **結尾：** 總結 Agent Builder 的新功能（如自訂工具、團隊工作區、多模型支援），並邀請所有使用者前往 `langchain.com` 免費試用。

## 關鍵見解
1.  **代理人而非工作流程**：強調其核心是自主的代理人，由提示、工具和記憶驅動，而非寫死的固定流程，使其更具彈性與智慧。
2.  **迭代式開發與記憶**：最亮眼的功能是在除錯模式中能即時糾正代理人，它會將回饋融入自己的系統提示中，實現真正的學習與進化。
3.  **校準與回饋循環**：高效的代理人是透過「校準搜尋」迭代而成的。先提供少量結果讓使用者給予回饋，再進行大規模搜索，確保最終結果的準確性。
4.  **代理人擁有檔案系統**：代理人可以讀寫檔案，這對於儲存複雜結果（如候選人列表）或跨對話記住上下文（如工作描述）至關重要，避免了在聊天介面中輸出大量雜亂資訊。
5.  **從模糊到具體**：展示了建構器如何引導使用者，將一個模糊的想法（「建立招募代理人」）轉化為一個可靠、具體的自動化工具。

## 精彩時刻
- **即時教導**：使用者在除錯模式中直接教導代理人：「你應該先問後續問題……記住這一點。」，而代理人也確實更新了自己的核心指令，展現了驚人的學習能力。
- **記憶的展現**：成熟的代理人在執行任務前，會先去檔案系統檢查「是否已經有儲存過的工作描述」，這完美體現了其跨對話的記憶能力。
- **聰明的產出**：當被要求尋找 50 位候選人時，代理人沒有在聊天中列出所有資料，而是聰明地回覆已將結果儲存至檔案，展示了高度的實用性。

# Langsmith Agent Builder: Public Beta Launch & Demo

## TL;DR
Langsmith has launched Agent Builder, a no-code platform for creating AI agents through a chat interface. These agents, equipped with tools, memory, and even sub-agents, can be tested and iteratively improved to handle complex tasks, turning vague ideas into reliable automation tools for everything from deep research to personal productivity.

## Story Flow
- **Beginning:** The video announces the public beta of Langsmith Agent Builder, positioning it as a true "agent builder" rather than a traditional "workflow builder." It's driven by the agent's own prompt, tools, and memory.
- **Middle:** The core of the video is a live demo creating a "LinkedIn Recruiter" agent from scratch.
    1.  **Creation & Clarification:** The user gives a vague prompt, and the builder proactively researches and asks several follow-up questions to clarify the agent's specific goals, triggers, and required data.
    2.  **Testing & Learning:** In "debug mode," the agent's first attempt is to search directly. The user corrects it on the spot, telling it, "You should ask followup questions to get a better sense of the JD. Remember this." The agent then updates its own system prompt, demonstrating true "memory."
    3.  **Real-World Application:** A mature agent is shown handling a recruiting task. It starts by asking for details, checks its file system for a pre-existing Job Description, runs a small "calibration search" to get user feedback, and only then executes the large-scale search for 50 candidates, saving the results to a file.
- **End:** The video summarizes new features of the public beta (like bring-your-own-tools, team workspaces, and multi-model support) and invites everyone to try it for free at `langchain.com`.

## Key Insights
1.  **Agent-centric, Not Workflow-driven:** The core concept is building an autonomous agent driven by a prompt, tools, and memory, not a rigid, predefined workflow. This makes it more flexible and intelligent.
2.  **Iterative Development with Memory:** The standout feature is the ability to correct the agent in debug mode and have it permanently update its own system prompt, allowing it to truly learn and evolve from feedback.
3.  **The Calibration Feedback Loop:** Effective agents are refined using a "calibration search"—getting a small sample of results, receiving user feedback, and then scaling up the search. This ensures the final output is accurate.
4.  **Agents Have a File System:** Agents can read and write to a dedicated file system. This is crucial for storing large outputs (like a list of candidates) or remembering context across sessions (like saved JDs), preventing chat clutter.
5.  **From Vague Idea to Specific Tool:** The builder itself guides the user from a fuzzy concept ("a recruiter agent") to a reliable, concrete automation tool by proactively asking clarifying questions.

## Notable Moments
- **Real-time Teaching:** The moment the user teaches the agent in debug mode: "You should ask followup questions... Remember this," and the agent proceeds to edit its own source instructions, showing a powerful learning capability.
- **Memory in Action:** The mature agent demonstrates its long-term memory by checking its file system for a "saved job description file" before starting a new search.
- **Intelligent Output:** When asked to find 50 candidates, the agent doesn't spam the chat. Instead, it smartly responds that it has saved the results to a file, showcasing its practical design.
