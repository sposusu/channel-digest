# Langmith Poly 發表：您的 AI 工程助理

## 重點摘要
Langsmith 推出了一款名為 Langmith Poly 的 AI 工程助理。這款工具內建於 Langsmith 平台，旨在透過分析對話、偵錯追蹤以及優化提示詞，幫助開發者簡化 AI 工程的工作流程。

## 故事大綱
- 開場：影片開頭宣布推出 Langmith Poly，一款專為 AI 工程師設計的 AI 助理。
- 中段：影片接著展示了 Poly 在 Langsmith 平台中的三種核心應用場景：
    1. **對話串分析 (Thread Analysis)**：使用者可以在對話串頁面中，向 Poly 提問關於整個對話的內容，例如「使用者在這段對話中是否曾感到沮ر喪？」，Poly 會分析對話並給出結論。
    2. **單次執行追蹤分析 (Trace Analysis)**：在單次的執行紀錄中，Poly 可以協助偵錯、找出 Agent 的錯誤或可改進之處。它甚至能連結外部文件，提供更豐富的背景資訊來回答問題。
    3. **遊樂場提示詞優化 (Playground Prompt Optimization)**：在遊樂場中，使用者可以要求 Poly 修改或優化系統提示詞。影片示範了如何命令 Poly 更新提示詞以限制 Agent 的搜尋次數，並透過並排比較功能，清楚地看到修改前後的差異與結果。
- 結尾：影片總結了 Poly 的三種使用方式，並宣布此功能目前處於 Beta 測試階段，鼓勵使用者試用並提供回饋。

## 關鍵見解
- **整合式助理**：Poly 直接整合在 Langsmith 平台內，讓 AI 工程師不需切換工具就能進行分析與偵錯。
- **多層次分析**：Poly 能從高層次的對話串（Threads）和深層次的單次執行追蹤（Traces）進行分析，滿足不同情境的需求。
- **主動的工程夥伴**：Poly 不僅僅是問答工具，更能根據指令主動修改提示詞，並透過比較功能驗證成效，成為提示詞工程的得力助手。
- **外部知識整合**：Poly 能搜尋並整合 LangChain 的官方文件等外部知識，使其分析與建議更具深度和準確性。
- **加速迭代循環**：透過即時分析、偵錯與優化建議，Poly 大幅縮短了從發現問題到解決問題的開發迭代時間。

## 精彩時刻
- **情感分析**：向 Poly 提問「使用者是否曾感到沮喪？」來分析對話，這是一個非常實用的洞察。
- **錯誤現形**：Poly 準確地指出了 Agent 在回應中出現的邏輯矛盾與錯誤陳述。
- **Git Diff 風格的比較**：在優化提示詞時，系統以類似 Git Diff 的視覺化方式，清晰呈現新舊版本的差異。
- **強化指令**：當第一次修改提示詞效果不彰時，Poly 能理解並「強化語氣」來產生更嚴格的指令，成功修正了 Agent 的行為。

# Introducing Langmith Poly: Your AI Engineering Assistant

## TL;DR
Langsmith has launched Langmith Poly, an AI engineering assistant integrated directly into the Langsmith platform. This tool is designed to streamline AI engineering workflows by helping developers analyze conversations, debug traces, and optimize prompts.

## Story Flow
- **Beginning**: The video introduces Langmith Poly, a new AI assistant purpose-built for AI engineering.
- **Middle**: The presentation demonstrates Poly's three core functionalities within the Langsmith platform:
    1.  **Thread Analysis**: In the thread view, users can ask Poly questions about an entire conversation, such as, "Did the user ever seem frustrated in this convo?" Poly analyzes the thread and provides a summary.
    2.  **Trace Analysis**: Within a specific run trace, Poly helps debug issues and identify where an agent made mistakes or could have performed better. It can also search external documentation to enrich its answers with more context.
    3.  **Playground Prompt Optimization**: In the playground, users can direct Poly to edit and optimize system prompts. The demo shows how Poly updates a prompt to limit an agent's search rounds and uses a side-by-side comparison feature to visualize the changes and their impact on the output.
- **End**: The video recaps the three primary uses of Poly, announces that the feature is in beta, and encourages users to try it out and provide feedback.

## Key Insights
- **Integrated Assistant**: Poly is built directly into the Langsmith platform, allowing AI engineers to analyze and debug without switching contexts.
- **Multi-Level Analysis**: Poly can operate at both a high-level (conversation threads) and a deep, granular level (individual traces), catering to different analytical needs.
- **Proactive Engineering Partner**: More than a Q&A bot, Poly actively participates in prompt engineering by editing prompts on command and verifying the impact through a comparison view.
- **External Knowledge Integration**: Poly can search and incorporate external knowledge, like LangChain's official documentation, to provide more insightful and accurate analysis.
- **Accelerated Iteration Loop**: By offering real-time analysis, debugging, and optimization suggestions, Poly significantly shortens the development cycle from identifying a problem to implementing a solution.

## Notable Moments
- **Sentiment Analysis**: Asking Poly "Did the user ever seem frustrated?" is a powerful and practical use case for understanding user interaction.
- **Catching Agent Flaws**: Poly accurately identifies an agent's logical contradictions and false statements in its response.
- **Git Diff-Style Comparison**: The visual, side-by-side comparison of prompt changes provides a clear and intuitive way to track optimizations.
- **Strengthening Instructions**: When the first prompt modification didn't work, Poly understood the need to "strengthen the language" of the instruction to successfully correct the agent's behavior.
